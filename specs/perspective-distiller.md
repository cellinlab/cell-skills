# 视角蒸馏 Skill 设计文档

## 1. 目标概述

这个 Skill 用于把一个公开人物、创作者、思想家、操盘手，或者一个主题方法论，蒸馏成一套可复用的「视角 Skill」。

它解决的不是“模仿某人的语气”，而是帮助用户完成下面这件事：

- 用公开材料提炼这个人是怎么想的，而不只是他说过什么
- 提炼他的心智模型、判断启发式、表达 DNA、反模式与诚实边界
- 把这些内容整理成一个可运行、可复查、可迭代的 Skill 包
- 让生成结果既能当思维顾问，也能在需要时进入更强的人物视角模式

## 2. 设计来源

这个 Skill 的方法论明显吸收了 `nuwa-skill` 的核心价值，但会做更适合 Cell 仓库的改造。

保留的部分：

- 不做语录拼贴，而是提炼认知操作系统
- 使用多源研究，而不是单一引用
- 通过三重验证判断什么才算真正的心智模型
- 显式保留矛盾、张力和诚实边界
- 最终产物是一个可独立使用的 Skill 包

改造的部分：

- 不强依赖「女娲」这类强人设叙事
- 不硬性要求 subagents，改成“六路研究工作流”，在工具允许时可并行
- 默认把结果做成更稳的“顾问型视角 Skill”，而不是一上来就做夸张角色扮演
- 更强调研究留痕与仓库内自包含结构

## 3. 技能定位

它是一个“视角蒸馏 / 思维框架提炼 / Perspective Skill Builder”。

它不是：

- 普通人物介绍 Skill
- 名言语录整理 Skill
- 单纯风格模仿 Skill

## 4. 与现有 Skill 的关系

- `perspective-distiller`
  - 负责生成或更新一个人物 / 主题视角 Skill
- `opc-case-research`
  - 负责系统化拆案例，不负责生成角色化视角 Skill
- `celf-style-writer`
  - 负责写法与风格，不负责蒸馏一个完整认知系统
- `learning-builder`
  - 负责学习路径设计，不负责把一个对象蒸成顾问 Skill

## 5. 建议技能名

建议英文目录名：

`perspective-distiller`

建议展示名：

`视角蒸馏`

## 6. 适用场景

这个 Skill 主要适用于：

- “帮我做一个保罗·格雷厄姆 / 芒格 / 乔布斯视角 Skill”
- “我想蒸馏某个人的思维方式，不只是模仿他说话”
- “把某个主题，比如 X 运营 / 独立开发 / 产品判断，蒸成一个方法论 Skill”
- “已有一个人物 Skill，帮我更新到最近材料并补充边界”

## 7. 非目标

V1 默认不解决以下问题：

- 非公开人物、隐私人物的深挖
- 用很薄的信息强行蒸馏出一个完整认知系统
- 把人物素材包装成绝对真实、绝对一致的“分身”
- 代替法律、医学、投资等高风险专业判断本身

## 8. 核心用户问题

这个 Skill 需要稳定回答以下问题：

1. 这个人真正稳定的思维框架是什么？
2. 哪些只是他说过的话，哪些才是他反复使用的判断镜片？
3. 如何在保留辨识度的同时，不把结果做成拙劣模仿秀？
4. 如何把研究结果沉淀成一个可继续更新的 Skill 包？
5. 哪些地方应该明确承认“蒸不出来”或“证据不足”？

## 9. 典型触发语句

- 蒸馏一个乔布斯视角
- 帮我做个芒格式的思维顾问 Skill
- 造一个某某人的 perspective skill
- 这个人物我已经有初版了，帮我更新
- 不要做人物介绍，要做成一个能用的思维框架 Skill

## 10. 默认输入约定

### 10.1 必填输入

- 至少提供以下之一：
  - 明确的人名
  - 明确的主题
  - 已有的旧 Skill 路径

### 10.2 选填输入

- 聚焦方向：创业 / 产品 / 写作 / 决策 / 教学 / 投资 / 人生策略
- 用途：顾问模式 / 角色模式 / 研究沉淀
- 用户提供的一手素材：书籍、字幕、transcript、长文、笔记
- 是否要更新已有 Skill

### 10.3 默认假设

如果用户没说明，默认：

- 输出语言为中文
- 优先公开材料与用户提供的一手素材
- 最终产物落成自包含 Skill 包
- 默认以“顾问型视角”优先，而非表演型角色扮演

## 11. 输出契约

### 11.1 标准输出

至少包含：

- 目标定义
- 研究来源结构
- 核心心智模型
- 决策启发式
- 表达 DNA
- 价值观 / 反模式 / 张力
- 诚实边界
- 来源附录

### 11.2 交付形态

默认交付为一个 Skill 目录，包含：

- `SKILL.md`
- `references/research/*.md`
- 必要的模板与脚本

## 12. 方法论原则

### 12.1 HOW 比 WHAT 更重要

蒸馏的是他看世界的方式，而不是台词本。

### 12.2 三重验证决定“是否纳入核心模型”

跨域复现、生成力、排他性，缺一不可。

### 12.3 公开材料越多，不等于可信信息越多

先看原始材料，再看二手解读。

### 12.4 矛盾不是 bug

人物的演化、领域差异和内在张力，往往恰恰是最有价值的部分。

### 12.5 诚实边界必须写出来

没有边界的视角 Skill 不可信。

## 14. V1 关键约束

1. 不拿名言摘抄冒充思维蒸馏
2. 不用单一来源生成完整人物模型
3. 不掩盖矛盾和信息不足
4. 必须生成可独立复制的 Skill 包
5. 必须保留原方法论中真正有价值的研究与验证机制
