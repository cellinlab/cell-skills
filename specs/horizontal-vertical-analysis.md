# 横纵研究工作台 Skill 设计文档

## 1. 目标概述

这个 Skill 用于把“帮我系统研究一下这个产品 / 公司 / 概念 / 人物”从资料堆砌，升级成一套可复用的双轴研究工作流：

- 纵向看生命历程
- 横向看当下竞争或替代格局
- 交汇后产出真正的新判断
- 默认沉淀成结构化研究报告，必要时导出 PDF

## 2. 设计来源

核心吸收 `khazix-skills/hv-analysis`。

保留的部分：

- 横纵分析法本身
- 一手来源优先的研究逻辑
- 竞品 A / B / C 三种场景判断
- “纵向叙事 + 横向比较 + 交汇洞察”的报告结构
- Markdown 转 PDF 的正式交付能力

改造的部分：

- 把原本很长的主 Skill 拆成 references，符合当前仓库的 progressive disclosure 结构
- 从“卡兹克文风”切成更适合 Cell 的研究型表达：更收敛、更强调事实边界和可复用输出
- 加入 `事实 / 推断 / 未知` 的默认证据契约
- 不把 PDF 作为唯一交付，改成 Markdown 母稿优先、PDF 可选

## 3. 技能定位

它是一个“通用深度研究与竞品分析” Skill。

它不是：

- 简单百科解释器
- 八卦型人物介绍器
- 法律、财务、投资尽调替代品
- 一上来就写长文、不先搭证据骨架的生成器

## 4. 核心原则

1. 先建研究骨架，再写判断
2. 纵轴必须讲因果，不讲流水账
3. 横轴必须讲真实差异，不讲参数堆表
4. 交汇段必须产出新的判断
5. 信息不足时，宁可收窄，不硬补
