# 小红书配图导演 Skill 设计文档

## 1. 目标概述

这个 Skill 用于把小红书配图，从“随手出几张图”，升级为“先提案、再出图、再审查”的设计工作流。

它主要回答：

- 当前内容适合什么视觉方向
- 先出哪 2-3 个提案最值得比较
- 什么情况下用 AI 出图，什么情况下用 HTML 精确渲染兜底
- 怎样保证图有小红书原生感，而不是模板感

## 2. 设计来源

核心吸收 `huashu-xhs-image`。

保留的部分：

- 先提案，后生成
- 3:4 竖版优先
- AI 生成与 HTML 兜底双路径
- 生成后必须做设计与平台吸引力双重审查

改造的部分：

- 从“花叔个人审美画像”改成更适合 Cell 仓库的设计画像
- 更强调与 `xhs-title-selector`、`content-strategy-diagnosis` 的协作
- 去掉特定私有脚本路径绑定

## 3. 核心原则

1. 不跳过设计提案
2. 先做信息层级和视觉逻辑，再做风格表面
3. 小红书图的第一任务是停留和可读，不是炫技
