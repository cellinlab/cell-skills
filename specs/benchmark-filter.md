# 对标筛选 Skill 设计文档

## 1. 目标概述

这个 Skill 用于帮助用户在创作者、超级个体、一人公司或内容型业务语境下，找到值得研究和模仿的对标，并判断一个已有对标到底值不值得继续学。

它要解决的不是“随便列几个同行名字”，而是帮助用户回答：

- 这个对标是真的有商业价值，还是只有表面热闹
- 我到底该模仿他的什么，而不是崇拜他的结果
- 这个案例适合拿来学获客、内容、产品、交付还是定位
- 现在该复制机制，还是该深入研究成一个完整案例

## 2. 设计来源

这个 Skill 主要吸收 `dbskill` 的 `dbs-benchmark`。

保留的部分：

- 对标首先看经营信号，而不是只看名气
- 看得懂商业主干，才谈得上模仿
- 模仿要有颗粒度，不是泛泛“学一下”
- 要压制“我不喜欢 / 不像我 / 不适合我”这类过早噪音

改造的部分：

- 不把“自我因素”完全删除，而是拆成两个阶段
  - 先做客观筛选
  - 再做资源与阶段适配
- 把对标范围收敛到更贴近 Cell 仓库的创作者 / 一人公司 / 内容业务语境
- 明确与 `opc-case-research` 的分工

## 3. 技能定位

它是一个“对标筛选与模仿拆解 Skill”，不是完整案例调研 Skill。

## 4. 与现有 Skill 的关系

- `benchmark-filter`
  - 负责找到值不值得学、该学什么、先学哪一层
- `opc-case-research`
  - 负责对某个已经选定的对象做系统化公开信息研究

理想分工：

1. 先用 `benchmark-filter` 确定谁值得研究、为什么值得研究
2. 再把高价值对象交给 `opc-case-research` 做深拆

## 5. 建议技能名

建议英文目录名：

`benchmark-filter`

建议展示名：

`对标筛选`

## 6. 适用场景

- 不知道该研究谁
- 手里有几个候选对象，不知道哪个更值
- 已经有对标，但模仿效果很差，想知道问题出在哪
- 想把“学一个人”拆成“学他的哪一层机制”

## 7. 非目标

V1 默认不解决以下问题：

- 自动做海量全网对标搜索
- 详细收入尽调
- 投资价值判断
- 没有任何方向时的无限发散

## 8. 核心用户问题

1. 这个人 / 这个业务值不值得成为我的对标？
2. 我是在模仿结果，还是在模仿机制？
3. 应该复制他的内容打法、产品模型，还是商业路径？
4. 我现在这个阶段适不适合直接学他？
5. 学不动到底是因为案例不对，还是因为模仿颗粒度太粗？

## 9. 典型触发语句

- 帮我找几个对标
- 这个人值不值得学
- 我应该模仿谁
- 我现在这个对标为什么学不动
- 先别深度调研，先帮我筛一下

## 10. 默认输入约定

### 10.1 必填输入

- 至少提供以下之一：
  - 候选对标对象
  - 自己想解决的问题
  - 想学的目标能力

### 10.2 选填输入

- 当前阶段：刚起步 / 已有收入 / 已有内容系统 / 想升级模型
- 重点能力：内容 / 产品 / 交付 / 渠道 / 定位 / 商业模式
- 是否已有现成对标

### 10.3 默认假设

如果用户没说明，默认：

- 输出语言为中文
- 先做客观筛选，再做阶段适配
- 重点看经营信号、可理解性和可复制机制
- 如果需要做深拆，再交给 `opc-case-research`

## 11. 输出契约

### 11.1 标准版

至少包含：

- 对标一句话判断
- 五层筛选结果
- 值得学的层
- 不值得学的层
- 第一模仿动作

### 11.2 候选列表模式

如果是多个候选对象，至少包含：

- 候选对比表
- 优先级排序
- 每个对象更适合学什么
- 推荐下一步

## 12. 方法论原则

### 12.1 先看经营信号，再看人设魅力

粉丝、声量和个人魅力都可能制造误判。

### 12.2 先看模型主干能否被理解

看不懂他怎么获客、转化、交付、复购，就谈不上模仿。

### 12.3 模仿颗粒度要足够细

不是“学他做内容”，而是“学他用什么形式、节奏、结构、产品入口、转化路径”。

### 12.4 先压住 ego 噪音，再讨论阶段适配

“不像我”不能比“能不能学会”更早出现。

### 12.5 深拆和筛选分开

筛选解决“值不值得研究”，深拆解决“研究完能得到什么”。

## 14. V1 关键约束

1. 不拿粉丝数替代经营信号
2. 不拿“适不适合我”当第一层判断
3. 必须明确“学什么”和“不学什么”
4. 必须提供一个可执行的第一模仿动作
5. 与 `opc-case-research` 的边界要清楚
