# 口播脚本打磨 Skill 设计文档

## 1. 目标概述

这个 Skill 用于把“写得通顺”的脚本，进一步打磨成“说得顺、录得顺、听得顺”的口播稿。

它主要回答：

- 为什么一读就卡
- 哪些句子太书面、太长、太像 AI
- 哪些地方要切句、停顿、重音、转口语
- 脚本是否适合真实录制节奏

## 2. 设计来源

核心吸收 `huashu-script-polish`。

保留的部分：

- 三遍审校思路：内容、口语化、演示 / 节奏标注
- “大声读一遍”作为判断标准
- 短句化、去书面腔、加自然连接词

改造的部分：

- 去掉个人语料绑定
- 更适合教程、口播、短视频、直播切片等泛场景

## 3. 核心原则

1. 口语化不是加口头禅，而是让句子可说
2. 一切修改都要服务于声音和节奏
3. 不牺牲信息密度去换假装自然
