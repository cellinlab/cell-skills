# 拆书素材提炼 Skill 设计文档

## 1. 目标概述

这个 Skill 用于把一本书、一个章节、若干摘录、读书笔记，或者与书相关的一组输入，压缩成适合后续内容创作使用的“写作素材包”。

它要解决的不是“帮用户写一篇读书笔记”，而是帮助用户完成下面这件事：

- 从一本书里提炼出真正可复用的观点
- 找出能承载论证与情绪的故事
- 把原书中的表达转成可用于写作的金句化素材
- 把这些颗粒重组为可继续深写的素材块
- 最后产出一篇可以直接交给 `skills/deep-writer/SKILL.md` 接力的素材文章

## 2. 设计来源

这个 Skill 的方法论参考了 `ljg-skill-xray-book` 这类“多轮认知压缩”的拆书思路，但产物方向明确做了改造。

保留的原则：

- 不是一次读完就下结论，而是通过分轮提炼投入更多认知算力
- 先抽骨架，再提结构，再做最终压缩
- 输出必须是结构化的，而不是随意感想

明确改造的部分：

- 不以“全书结构图”或“完整拆书报告”为终点
- 不以 Org 报告或 ASCII 图为主要交付物
- 重点从“理解一本书”切换到“提炼可写材料”
- 最终产物面向内容创作，而不是面向阅读记录

## 3. 适用场景

这个 Skill 主要面向下面几类场景：

- 拆一本书，给公众号 / 专栏 / 视频脚本准备素材
- 把读书笔记压成可直接调用的观点、故事与金句
- 为某个主题找一本书里的高密度论证和案例
- 先做拆书素材包，再交给 deep-writer 继续深化成长文
- 从一本书里提炼多个可复用的选题角度和段落模块

## 4. 非目标

V1 不解决以下问题：

- 学术型文献综述或严谨引文整理
- 完整书摘库建设
- 对整本书逐章逐节做百科式摘要
- 长篇原文摘抄或大段引用整理
- 代替最终文章写作本身

## 5. 建议技能名

建议英文目录名：

`book-material-miner`

理由：

- 比 `book-xray` 更偏向“提炼素材”，不和参考项目过度同构
- 保留“挖掘”意味，符合从书中采矿的直觉
- 名称足够短，适合长期引用

建议展示名：

`拆书素材提炼`

## 6. 核心用户问题

这个 Skill 需要稳定回答以下问题：

1. 这本书里最值得拿去写内容的几个核心判断是什么？
2. 哪些内容更适合做故事、案例、实验片段或场景引子？
3. 哪些表达适合改造成金句，而不是生硬摘抄？
4. 怎样把零散颗粒拼成未来可直接调用的素材块？
5. 怎样输出一篇足够清晰、但还保留深化空间的素材文章给 deep-writer？

## 7. 典型触发语句

- 帮我把这本书拆成写作素材，不要做普通摘要
- 从这本书里提炼观点、故事、金句和可复用素材块
- 这是一堆读书笔记，帮我整理成后面能写文章用的素材
- 我想围绕这本书写长文，先给我一份适合 deep-writer 接力的素材包
- 不要直接写成品，先把书拆成未来可写的结构和段落模块

## 8. 默认输入约定

### 8.1 必填输入

- 书名、摘录、读书笔记、章节摘要、书籍链接中的至少一种

### 8.2 选填输入

- 目标主题
- 目标读者
- 发布场景
- 产物重点：观点 / 故事 / 金句 / 素材文章
- 输出深度：快速版 / 标准版 / 深度版

### 8.3 默认假设

如果用户没说明，默认：

- 输出语言为中文
- 交付的是素材包，不是读后感
- 优先做金句化改写，不大段摘抄原文
- 要显式区分原书观点与后续外推
- 最终最好能交给 deep-writer 继续做深度写作

## 9. 输出契约

### 9.1 快速版

至少包含：

- 拆书定位卡
- 3-5 条观点
- 1-3 个故事或案例
- 3-5 条金句化表达
- 一个简版素材文章方向

### 9.2 标准版

至少包含：

- 拆书定位卡
- 观点库
- 故事库
- 金句库
- 4-8 个素材块
- 一篇可交给 deep-writer 的素材文章
- 交接说明

### 9.3 深度版

在标准版基础上尽量补齐：

- 更清晰的概念块与对比块
- 更多边界条件或反例
- 更稳的文章主线与段落职责
- 更明确的 deep-writer 接力建议

## 10. 方法论原则

### 10.1 先找“写什么”，再找“讲了什么”

不是先复述全书，再想怎么用；而是先定位哪些内容值得写，再反过来组织材料。

### 10.2 分轮加压，而不是一次粗暴总结

默认采用三轮：

- 主题与张力扫描
- 素材颗粒提炼
- 写作重组

### 10.3 强调素材复用性

每个强素材都要尽量回答：

- 适合写在哪类文章里
- 适合放在文章哪个位置
- 它能承担什么写作任务

### 10.4 最终产物服务于深写

素材文章不是终稿，而是为 `deep-writer` 准备的一篇高密度中间底稿。

## 11. 与参考项目的差异化

这个 Skill 应明确区别于参考项目：

- 参考项目偏“认知压缩后的完整拆书报告”
- 本 Skill 偏“为内容创作准备的写作素材包”

具体差异：

- 参考项目强调骨架、论证链、灵魂提取、餐巾纸压缩
- 本 Skill 强调观点库、故事库、金句库、素材块和素材文章
- 参考项目的终点是理解与压缩
- 本 Skill 的终点是未来还能继续写

## 13. V1 关键约束

1. 不能用章节摘要冒充素材提炼
2. 不能大量照抄原书
3. 不能把推测冒充原书观点
4. 必须告诉用户每类素材的可用场景
5. 最终产物必须包含面向 deep-writer 的交接意识
