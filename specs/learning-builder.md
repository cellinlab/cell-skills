# learning-builder 设计文档

## 1. 目标

把“想学 X”这类模糊请求，转成可执行学习包：

- 有学习者画像
- 有权威来源优先研究
- 有阶段任务与练习
- 有里程碑与复盘
- 能衔接内容创作输出

## 2. 适用场景

- 用户想系统学习一个新主题
- 用户想在有限时间内搭建学习路径
- 用户希望边学边产出（文章/脚本/讲义/项目）

## 3. 不适用场景

- 一次性问答
- 纯知识科普写作
- 只要网页设计、不需要学习包

## 4. 输入

- 学习主题
- 当前基础
- 目标结果
- 时间预算
- 输出场景
- 来源偏好与限制

## 5. 输出

- 学习包主文档（Markdown）
- 来源附录（URL + 日期 + 可信理由）
- 可选 docx / pdf 导出建议

## 6. 核心流程

1. 学习者画像 intake
2. Authority-first research
3. 学习包组装
4. 阶段练习与里程碑校验
5. 30 天复盘与输出计划

## 7. 质量标准

- 每阶段都有动作和产物
- 关键结论有可追溯来源
- 学习包可执行，不是知识堆砌
- 与用户目标和时间约束一致
