# 内容策略诊断 Skill 设计文档

## 1. 目标概述

这个 Skill 用于在内容真正开写之前，先判断一个选题、草稿、内容计划或渠道想法，到底该怎么做才更像一个成立的内容产品。

它解决的不是“帮用户把内容直接写出来”，而是帮助用户先回答下面这些更关键的问题：

- 这个题值不值得做
- 它更适合什么形式和平台
- 核心角度有没有打中用户
- 材料密度够不够支撑表达
- 如果内容带商业目标，它和产品 / 服务 / 业务是否连得上

## 2. 设计来源

这个 Skill 的方法论明显吸收了 `dbskill` 里的 `dbs-content`，但会做更适合 Cell 仓库的改造。

保留的部分：

- 先诊断再写，不拿生成代替判断
- 内容形式要和题材特性匹配
- 标题 / 封面 / 开头的重要性必须被看见
- 商业目标存在时，要检查内容和产品之间有没有真正的连接

改造的部分：

- 不把所有内容都粗暴收敛成“卖货前置动作”
- 不把平台规则说成固定真理，而是强调场景匹配
- 不沿用过强的人设化表达，而是改成 Cell 更稳、更可复用的分析框架
- 明确和 `deep-writer`、`celf-style-writer`、`hook-optimizer` 的分工

## 3. 技能定位

它是一个“内容前置诊断 Skill”，不是一个“写作 Skill”。

更准确地说，它解决的是：

- 写之前，先判断该不该写、怎么写、写给谁、在哪发
- 用户已经有内容方向，但还没把形式、角度、材料和目标对齐
- 用户需要一个编辑 / 内容策略顾问视角，而不是代笔视角

## 4. 与现有 Skill 的关系

- `deep-writer`
  - 负责深度内容的 Brief、结构和正文生成
- `celf-style-writer`
  - 负责语言风格、活人感和 Cell 气质
- `hook-optimizer`
  - 负责短内容开头与前几秒吸引力
- `content-strategy-diagnosis`
  - 负责在这些动作之前，先判断内容策略本身对不对

理想顺序通常是：

1. 先用 `content-strategy-diagnosis` 判断方向、形式、平台和材料密度
2. 需要长内容时交给 `deep-writer`
3. 需要 Cell 风格时交给 `celf-style-writer`
4. 需要短视频开头时交给 `hook-optimizer`

## 5. 建议技能名

建议英文目录名：

`content-strategy-diagnosis`

建议展示名：

`内容策略诊断`

## 6. 适用场景

这个 Skill 主要适用于：

- 一个选题已经有了，但不知道该做成短视频、图文、文章还是直播
- 有一个初稿，但不确定问题到底出在内容本身、角度、形式还是表达效率
- 想做一组内容，但不知道该先打什么角度、补什么材料
- 想把内容和业务 / 产品 / 服务更稳地连起来
- 想知道“这条内容第一步该做什么”，而不是听一堆泛建议

## 7. 非目标

V1 默认不解决以下问题：

- 直接代替用户写完整内容
- 空口判断“这条内容一定会爆”
- 完整平台运营打法或投流策略
- 纯审美层面的封面设计执行
- 没有任何内容输入时的无边界选题发散

## 8. 核心用户问题

这个 Skill 需要稳定回答以下问题：

1. 这个选题真正值得打的点是什么？
2. 现在的表达形式是不是选错了？
3. 这条内容的问题更像是材料不够、角度不准，还是表达效率太低？
4. 如果目标是增长、转化、建立认知，打法分别应该怎么调？
5. 在不急着重写全文的前提下，第一步最该动哪里？

## 9. 典型触发语句

- 帮我看看这个内容该怎么做
- 这个题做成短视频还是文章更好
- 我有一个选题，但总觉得不成立
- 这篇稿子问题出在哪
- 不要直接帮我写，先帮我判断内容策略

## 10. 默认输入约定

### 10.1 必填输入

- 至少提供以下之一：
  - 选题
  - 内容计划
  - 初稿
  - 内容目标

### 10.2 选填输入

- 目标平台
- 目标读者
- 目标动作：涨粉 / 建认知 / 转化 / 成交 / 测试表达
- 当前已有材料
- 当前已有产品、服务、咨询或商业入口

### 10.3 默认假设

如果用户没特别说明，默认：

- 输出语言为中文
- 先判断内容策略，再决定是否进入写作
- 先找最主要的一个问题，不做漫天撒网式建议
- 如果内容带商业目标，就检查产品 / 服务连接
- 如果内容不带商业目标，不强行要求“先有产品后有内容”

## 11. 输出契约

### 11.1 快速版

至少包含：

- 内容一句话判断
- 推荐形式与平台
- 最主要的问题
- 第一行动建议

### 11.2 标准版

至少包含：

- 内容诊断卡
- 推荐形式 / 平台
- 五维诊断结论
- 主要风险与缺口
- 优先级最高的修改动作
- 适合交接的后续 Skill

### 11.3 深度版

在标准版基础上尽量补齐：

- 内容目标与用户动作路径
- 标题 / 封面 / 开头的策略建议
- 素材缺口清单
- 商业连接判断
- 未来内容矩阵或延展方向

## 12. 方法论原则

### 12.1 先判断“内容产品”是否成立，再谈写法

如果内容本身没有抓手、材料或目标，优化标题和文风只是局部修补。

### 12.2 形式不是附件，而是策略的一部分

同一个题做成短视频、图文或长文，成立条件完全不同。

### 12.3 不把“表达问题”误判成“选题问题”

有些内容不是题差，而是没有找到正确载体、开头或结构。

### 12.4 商业目标存在时，检查内容和业务连接

如果用户希望内容服务于成交、咨询、私域或产品，就必须判断转化入口是否存在；如果用户当前只做品牌积累，也要如实说明。

### 12.5 动作建议必须具体

不说“你可以优化一下”，而是指出“先补哪类材料”或“先改哪个环节”。

## 14. V1 关键约束

1. 不直接代替用户把全文写完
2. 不用“能火 / 不能火”当核心结论
3. 不在没有商业目标时硬塞商业判断
4. 必须明确推荐形式和第一步动作
5. 必须能和现有写作类 Skill 顺畅衔接
